# Leave Management

A streamlined leave request and approval system that empowers employees to submit time-off requests while giving managers visibility and control over team availability.

## Overview

Employees submit leave requests specifying dates, type (Casual, Sick, Vacation), and reason. Managers receive requests from their direct reports, can approve or reject with a single click, and the employee receives instant notification of the decision. A real-time dashboard keeps managers informed of pending requests.

## Roles

### Employee
- Submit new leave requests with date range, type, and reason
- View status of all their submitted requests (Pending, Approved, Rejected)
- Receive notifications when requests are approved or rejected

### Manager
- View all pending leave requests from their team
- Approve or reject requests with optional comments
- Access dashboard showing team leave overview

## Key Scenarios

1. **Submit Leave Request**: Employee fills out leave form with dates and reason, submits for manager approval
2. **Review & Approve**: Manager sees pending request in dashboard, approves it, employee gets notified
3. **Reject with Feedback**: Manager declines request, provides reason, employee is notified
4. **Track Request Status**: Employee checks their request list to see current status

## Design Direction

**Tone**: Clean, professional, and calming — this is a workplace tool that should feel trustworthy and efficient.

**Palette**: Soft teals and warm neutrals with status-driven accents — green for approved, amber for pending, subtle red for rejected. Light mode primary with a refined, airy feel.

**Typography**: Modern sans-serif with clear hierarchy — readable labels, prominent status indicators.

**Visual Identity**: Minimal chrome, generous whitespace, card-based layouts. Status badges should be immediately scannable. Forms should feel lightweight and fast to complete.
